Interface Principale

Cette application a pour objectif de faciliter le redimensionnement et l'optimisation des images. Son interface est pratiquement toute dans la même fenêtre et les options sont simples. L'interface est divisée en 6 parties, indiquées par un numéro de 1 à 6. Il suffit de trouver celle qui vous intéresse.


1- Boîte de fichiers

Faites glisser les fichiers d'images pris en charge sur la boîte pour les traiter. L'application indiquera l'acceptation en animant les bords. Déposez les fichiers d'images que vous souhaitez optimiser à l'intérieur ou, alternativement, cliquez sur le bouton pour charger des fichiers en utilisant la boîte de dialogue standard de votre système. Les fichiers qui ne sont pas des images ou de formats non pris en charge seront affichés avec un message d'erreur à l'écran suivant, mais le traitement ne sera pas interrompu.


2- Réglages des images

Il y a 5 réglages possibles décrits ci-dessous :

  • Compression

    Définit d'où viendra la compression des images.

    • Tinify lorsque possible : Utilisera toujours le service de Tinipng si vous êtes en ligne, avez une clé API et que le type d'image est pris en charge. Sinon, ce sera local.
    • Meilleur résultat : Déclenchera 2 routines de compression, une pour Tinypng et une autre pour le local, à condition que Tinypng remplisse les conditions ci-dessus. Sinon, ce sera uniquement local. À la fin, l'image avec la plus petite taille sera utilisée.
    • Toujours local : Utilisera toujours le moteur interne de l'application pour effectuer toutes les activités.

  • Couleur de fond

    Définir la couleur de fond pour les conversions vers des formats sans canal alpha (transparence). Actuellement, seulement JPG. Choisissez la couleur transparente (damier) pour profiter du canal alpha. Si la couleur choisie est transparente et que l'image cible a un canal alpha et est convertie en JPG, le noir sera utilisé comme couleur de fond.


  • Conversion

    Définir le format vers lequel l'image résultante sera convertie. Si aucun n'est choisi, l'image reste dans son format original. Les exceptions sont BMP, PCX et HEIC qui sont automatiquement convertis en un autre format selon la configuration si un format spécifique n'est pas choisi dans ce réglage.


  • Fichiers

    Définir comment la nouvelle image sera créée en fichier. Les options sont :

    • Écraser : Le fichier original sera écrasé.
    • Créer un dossier : Un dossier appelé optimisé sera créé, s'il n'existe pas déjà, et le fichier y sera créé avec le nom original. Si un fichier du même nom existe déjà dans ce dossier, il sera écrasé.
    • Nouveau fichier : Un nouveau fichier avec le suffixe "-optimisé" sera créé.
    • Copie de sauvegarde : Fait une copie de sauvegarde du fichier original avec le suffixe "-backup" avant de l'écraser.

  • Dimensions

    Définir les dimensions de l'image en cours de traitement.

    • Ne pas modifier : L'image ne subira aucun redimensionnement.
    • Escalader : L'image est redimensionnée sur l'un des axes. Soit la largeur, soit la hauteur. Le choix apparaît lorsque cette option est sélectionnée. La nouvelle taille en pixels et l'axe auquel elle se réfère. La taille de l'autre axe est calculée de manière à conserver la même proportion de l'image.
    • Adapter : L'image sera redimensionnée pour s'adapter à l'intérieur d'un rectangle dont les dimensions, largeur et hauteur, sont demandées lorsque cette option est sélectionnée. La nouvelle taille sera la plus grande possible pour s'adapter aux dimensions choisies.
    • Couvrir : L'image sera redimensionnée pour couvrir le rectangle dont les dimensions, largeur et hauteur, sont demandées lorsque cette option est sélectionnée. La nouvelle taille sera la plus petite possible pour couvrir les dimensions choisies. Lorsqu'elle est utilisée dans Tinypng, l'IA est utilisée pour déterminer où couper l'image.
    • Miniature : Sur place, c'est la même chose que Couvrir. Dans Tinypng, il est capable d'étirer les arrière-plans pour créer des miniatures. Comme pour couvrir, la largeur et la hauteur sont demandées.


3- Barre de Commandes

Les 3 premiers boutons permettent de cocher et décocher des options relatives aux métadonnées de l'image. Ce sont, dans l'ordre : Informations sur le copyright, la géolocalisation et la date de l'image.

Les 3 derniers, également dans l'ordre, changent la langue de l'interface, font un don à l'auteur et invoquent l'écran de Configuration.


4- Compteurs

Les optimisations effectuées par cette application sont comptabilisées dans cet espace. Tant via Tinypng que localement. Un total général et un autre mensuel sont maintenus.

La comptabilisation mensuelle, tant locale qu'en ligne (Tinify), concerne uniquement les compressions effectuées à l'aide de cette application. Chaque image compte comme une compression, indépendamment des conversions et/ou redimensionnements. Pour voir combien de compressions vous avez effectuées sur Tinify au cours du mois en cours, activez "Afficher le comptage mensuel des compressions de Tinify" dans les paramètres.


5- Totaux

Affiche le total économisé et le nombre d'optimisations effectuées. Le pourcentage se réfère au total économisé par rapport à la taille avant les optimisations.


6- Pied de page

Ici, nous avons un indicateur de connexion à Internet, qui tente de vérifier en temps réel si cette connexion existe. Si elle n'existe pas, toutes les activités se feront via le moteur interne de l'application (local).

Nous avons également un menu avec 4 liens :

  • Confidentialité : Ouvre une page avec notre politique de confidentialité.
  • EULA : Ouvre une page avec notre licence d'utilisation.
  • À propos : Ouvre une boîte sur cette application et son auteur. Vous y trouverez des contacts, d'autres applications et diverses informations.

Conseils et observations

  • Les fichiers compressés envoyés dans le dossier optimisé utiliseront ce dossier s'il existe déjà et écraseront les fichiers portant le même nom à l'intérieur.
  • Les fichiers compressés avec un suffixe conserveront ce suffixe même si leur extension change (par exemple, s'ils changent également de format).
  • L'orientation automatique résout les problèmes d'images à l'envers présentant une donnée EXIF ​​telle que : Orientation : Rotation 180°. Cette commande suivra cette donnée EXIF ​​lorsqu'elle est présente. Notez que toutes les images de ce type ne sont pas à l'envers. Tout logiciel prenant en compte cette donnée EXIF ​​affichera l'image correctement, mais ce n'est pas le cas de tous.
  • Lire un son à la fin du traitement est utile si vous traitez plusieurs images et que vous vous absentez pour une autre tâche. Ainsi, un signal sonore vous avertit de la fin du traitement.
  • Le nombre mensuel de compressions, locales et en ligne (Tinypng), concerne uniquement les compressions effectuées avec cette application. Chaque image est comptabilisée comme une compression, qu'il y ait ou non des conversions et/ou des redimensionnements. Pour connaître votre consommation mensuelle de votre quota Tinypng, activez l'option « Afficher le nombre de compressions mensuelles Tinypng ».
  • Lorsque le fichier source est un SVG, aucune économie n'est calculée, car il est impossible de comparer des images vectorielles et des images matricielles, et les conversions ne génèrent jamais d'images plus petites. Les économies ne sont calculées que pour les images raster.
  • La couleur d'arrière-plan peut servir à :
    • Transformer l'arrière-plan transparent d'une image en un arrière-plan opaque de la couleur spécifiée.
    • Fournir une couleur d'arrière-plan pour la conversion d'une image avec transparence (canal alpha) vers une image qui n'en possède pas.
      • Si aucune couleur n'est spécifiée (image conservée transparente), le noir (RVB 0, 0, 0) sera utilisé.
  • Il n'est pas possible de remplacer l'arrière-plan opaque d'une image par un autre d'une couleur différente.


Interface de résultats

Après que les images sont glissées ou ouvertes et que le processus de compression, redimensionnement et conversion est lancé, l'écran ci-dessous apparaît. Vous y suivez simplement ce processus. L'interface est divisée en 6 parties, indiquées par un numéro de 1 à 6. Il suffit de trouver celle qui vous intéresse.


1- Liste des images traitées

Contient les images en cours de traitement pour votre vérification et suivi. Inclut les champs suivants :

  • Une miniature, qui montre souvent seulement le centre de l'image originale.
  • Le nom du fichier.
  • Indicateur de conversion avec le format source et destination.
  • Une icône désignant si le traitement est local ou distant.
  • Taille originale, compressée et pourcentage de compression.
  • Temps passé sur le traitement.
  • Statut du processus :
    • Traitement - animation
    • Terminé avec succès - icône verte
    • Une erreur s'est produite - icône rouge

2- Indicateur de traitement

Indique visuellement et par pourcentage combien du traitement a déjà été complété.


3- Compteur d'images

Indique combien d'images sont en cours de traitement et combien sont déjà terminées.


4- Temps de traitement

Indique le temps total du traitement de toutes les images.


5- Actions

Pour le moment, seulement le bouton pour revenir à l'interface principale. Rien n'empêche que vous glissiez de nouvelles images à traiter sur cette interface de résultat. L'application indiquera l'acceptation et les traitera de la même manière.


6- Indicateur de traitement de Tinypng.com

S'il y a une clé API valide et qu'elle est activée dans les paramètres, montre combien de compressions ont été faites dans le mois avec cette clé. Ainsi, il est facile de contrôler combien de compressions ont déjà été faites.



Paramètres

Ajustez ici les paramètres de l'application. L'interface est divisée en 4 parties, indiquées par un numéro de 1 à 4. Il suffit de trouver celle qui vous intéresse.


1- Clé API de Tinypng

Si vous choisissez d'ouvrir un compte sur Tinypng pour accéder à sa puissante API de compression et de redimensionnement d'images, entrez votre clé API dans cette zone de texte.


2- Conversions forcées

Les formats BMP, PCX et HEIC sont uniquement pris en charge pour la lecture. Par conséquent, pour être compressés et/ou redimensionnés, ils doivent obligatoirement être convertis dans un autre format. Ici, vous choisissez ce format par défaut. Notez que si vous optez pour un autre format de conversion dans le sélecteur de l'interface principale, il aura priorité sur ce paramètre.


3- Autres paramètres

  • Toujours au-dessus - Maintient la fenêtre de l'application toujours au-dessus des autres.
  • Signal sonore : Émet un signal sonore à la fin du traitement.
  • Comptage mensuel de Tinypng : Affiche dans l'interface le nombre de compressions effectuées avec le service Tinypng au cours du mois en cours.
  • Correction d'orientation : Tente de corriger l'orientation d'une image en fonction de son EXIF, si disponible et uniquement pour les formats pris en charge.
  • Réinitialiser tous les compteurs : Réinitialise les compteurs de compressions effectuées par l'application et recommence à zéro.

4- Boutons de commande

Enregistre les paramètres ou revient sans enregistrer.
