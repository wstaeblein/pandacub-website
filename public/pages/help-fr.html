Cette application a pour objectif de faciliter le redimensionnement et l'optimisation des images. Son interface est pratiquement toute dans la même fenêtre et les options sont simples. L'interface est divisée en 6 parties, indiquées par un numéro de 1 à 6. Il suffit de trouver celle qui vous intéresse.

1- Boîte de fichiers

Déposez les fichiers d'images que vous souhaitez optimiser à l'intérieur ou cliquez sur le bouton pour charger des fichiers en utilisant la boîte de dialogue standard de votre système.


2- Réglages des images

Il y a 5 réglages possibles décrits ci-dessous :

  • Compression

    Définit d'où viendra la compression des images.

    • Tinify lorsque possible : Utilisera toujours le service de Tinipng si vous êtes en ligne, avez une clé API et que le type d'image est pris en charge. Sinon, ce sera local.
    • Meilleur résultat : Déclenchera 2 routines de compression, une pour Tinypng et une autre pour le local, à condition que Tinypng remplisse les conditions ci-dessus. Sinon, ce sera uniquement local. À la fin, l'image de plus petite taille sera utilisée.
    • Toujours local : Utilisera toujours le moteur interne de l'application pour effectuer toutes les activités.

  • Couleur de fond

    Définit la couleur de fond pour les conversions vers des formats sans canal alpha (transparence). Actuellement, seulement JPG. Choisissez la couleur transparente (damier) pour profiter du canal alpha. Si la couleur choisie est transparente et que l'image de destination a un canal alpha et est convertie en JPG, le noir sera utilisé comme couleur de fond.


  • Conversion

    Définit le format dans lequel l'image résultante sera convertie. Si aucun n'est choisi, l'image reste dans son format original. Les exceptions sont BMP, PCX et HEIC qui sont automatiquement convertis dans un autre format selon la configuration si un format spécifique n'est pas choisi dans ce réglage.


  • Fichiers

    Définit comment la nouvelle image sera créée en fichier. Les options sont :

    • Écraser : Le fichier original sera écrasé.
    • Créer un dossier : Un dossier appelé optimisé sera créé, s'il n'existe pas déjà, et le fichier y sera créé avec le nom original. Si un fichier du même nom existe déjà dans ce dossier, il sera écrasé.
    • Nouveau fichier : Un nouveau fichier avec le suffixe "-optimisé" sera créé.
    • Copie de sauvegarde : Fait une copie de sauvegarde du fichier original avec le suffixe "-backup" avant de l'écraser.

  • Dimensions

    Définit les dimensions de l'image en cours de traitement.

    • Ne pas modifier : L'image ne subira aucun redimensionnement.
    • Échelle : L'image est redimensionnée sur l'un des axes. Soit la largeur, soit la hauteur. Le choix apparaît lorsque cette option est sélectionnée. La nouvelle taille en pixels et à quel axe elle se réfère. La taille de l'autre axe est calculée de manière à garder l'image avec les mêmes proportions.
    • Ajuster : L'image sera redimensionnée pour s'adapter à un rectangle dont les dimensions, largeur et hauteur, sont demandées lorsque cette option est sélectionnée. La nouvelle taille sera la plus grande possible pour s'adapter aux dimensions choisies.
    • Couverture : L'image sera redimensionnée pour couvrir le rectangle dont les dimensions, largeur et hauteur, sont demandées lorsque cette option est sélectionnée. La nouvelle taille sera la plus petite possible pour couvrir les dimensions choisies. Lorsqu'elle est utilisée dans Tinypng, l'IA est utilisée pour déterminer où couper l'image.
    • Miniature : Sur place, c'est la même chose que Couverture. Dans Tinypng, il est capable d'étirer les arrière-plans pour créer des miniatures. Comme pour la couverture, la largeur et la hauteur sont demandées.


3- Barre de Commandes

Les 3 premiers boutons permettent de cocher et décocher des options concernant les métadonnées de l'image. Ce sont, dans l'ordre : Informations sur le copyright, la géolocalisation et la date de l'image.

Les 3 derniers, également dans l'ordre, changent la langue de l'interface, font un don à l'auteur et invoquent l'écran de Configuration.


4- Compteurs

Les optimisations effectuées par cette application sont comptabilisées dans cet espace. Tant via Tinypng que localement. Un total général et un autre mensuel sont maintenus.

Le décompte mensuel, tant local qu'en ligne (Tinify), concerne uniquement les compressions effectuées à l'aide de cette application. Chaque image compte comme une compression indépendamment des conversions et/ou redimensionnements. Pour voir combien de compressions vous avez effectuées sur Tinify au cours du mois en cours, activez "Afficher le décompte mensuel des compressions de Tinify" dans les paramètres.


5- Totaux

Affiche le total économisé et le nombre d'optimisations effectuées. Le pourcentage se réfère au total économisé par rapport à la taille avant les optimisations.
