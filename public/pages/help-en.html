Main Interface

This app aims to facilitate the resizing and optimization of images. Its interface is almost entirely in the same window and the options are simple. The interface is divided into 6 parts, indicated by a number from 1 to 6. Just find the one that interests you.


1- File Box

Drag supported image files onto the box to process them. The app will indicate acceptance by animating the borders. Drop the image files you want to optimize inside or, alternatively, click the button to load files using your system's standard dialog box. Files that are not images or in unsupported formats will be shown with an error message on the next screen, but processing will not be interrupted.


2- Image Adjustments

There are 5 possible adjustments described below:

  • Compression

    Defines where the image compression will come from.

    • Tinify when possible: Will always use the Tinipng service if it is online, has an API key, and the image type is supported. Otherwise, it will be local.
    • Best result: It will trigger 2 compression routines, one for Tinypng and another for the local application, provided Tinypng meets the above requirements. Otherwise, it will only be local. In the end, the image with the smallest size will be used.
    • Always local: It will always use the application's internal engine to perform all activities.

  • Background color

    Defines the background color for conversions to formats without an alpha channel (transparency). Currently only JPG. Choose the transparent (checkered) color to take advantage of the alpha channel. If the chosen color is transparent and the destination image has an alpha channel and is converted to JPG, black will be used as the background color.


  • Conversion

    Defines the format to which the resulting image will be converted. If none is chosen, the image remains in its original format. The exceptions are BMP, PCX, and HEIC, which are automatically converted to another format according to the configuration if a specific format is not chosen in this setting.


  • Files

    Defines how the new image will be created in a file. The options are:

    • Overwrite: The original file will be overwritten.
    • Create folder: A folder called optimized will be created, if it does not already exist, and the file will be created in it with the original name. If a file with the same name already exists in this folder, it will be overwritten.
    • New file: A new file with the suffix "-optimized" will be created.
    • Backup copy: Creates a backup copy of the original file with the "-backup" suffix before overwriting it.

  • Dimensions

    Defines the dimensions of the image being processed.

    • Do not change: The image will not be resized.
    • Scale: The image is resized along one of the axes. Either width or height. The choice appears when this option is selected. The new size in pixels and which axis it refers to. The size of the other axis is calculated in such a way as to leave the image with the same aspect ratio.
    • Fit: The image will be resized to fit within a rectangle whose dimensions, width and height, are specified when this option is selected. The new size will be the largest possible to fit the chosen dimensions.
    • Cover: The image will be resized to cover the rectangle whose dimensions, width and height, are requested when this option is selected. The new size will be the smallest possible to cover the chosen dimensions. When used in Tinypng, AI is used to determine where to crop the image.
    • Miniature: On-site is the same as Cover. TinyPNG is able to stretch backgrounds to create thumbnails. To cover it, the width and height are specified.


3- Command Bar

The first 3 buttons allow you to select and unselect options related to image metadata. They are, in order: Copyright information, geolocation information, and image date.

The last 3, also in order, change the interface language, make a donation to the author, and bring up the Settings screen.


4- Counters

The optimizations made by this application are tracked in this space. Both via Tinypng and locally. A grand total and a monthly total are maintained.

The monthly count, both locally and online (Tinify), only includes compressions made using this app. Each image counts as compression regardless of whether there are conversions and/or resizing. To see how many compressions you've done in Tinify this month, enable "Show monthly Tinify compression count" in settings.


5- Totals

It shows the total savings and the number of optimizations made. The percentage refers to the total savings compared to the size before optimizations.


6- Footer

Here we have an internet connection indicator, which attempts to measure in real time whether that connection exists. If none of the activities exist, they will be handled by the application's internal (local) engine.

We also have a menu with 4 links:

  • Privacy: Access our privacy policy page.
  • EULA: Invokes page with our usage license.
  • About: Invokes a box about this application and its author. Here you can find contacts, other apps, and various data.
  • Help: This page.

Tips and Observations

  • Compressed files that go to the optimized folder will use the folder if it already exists and will overwrite files with the same name inside the folder.
  • Compressed files with a suffix will receive the suffix even if they change their extension (e.g., they are also changing format).
  • Auto-orient solves problems such as images that are upside down but have an EXIF like: Orientation: Rotate 180. This command will follow the EXIF whenever it exists. Note that not all images like this appear upside down. Any software that takes this EXIF into account will display the image correctly, but not all are like that.
  • Playing the sound when processing is finished is useful if you put several images to process and go do something else. This way the sound alerts you that the processing has finished.
  • The monthly count, both local and online (Tinypng), is only for compressions done using this app.
    • Each image counts as a compression regardless of whether there are conversions and/or resizings. To see exactly how much of your monthly Tinypng quota you've used, enable "Show Tinypng monthly compression count".
  • When the source file is an SVG, no savings are calculated because there's no way to compare vector images with raster images, and because conversions never generate smaller images. Savings are only calculated between raster images.
  • Background color can be used to:
    • Transform the transparent background of an image into an opaque background of the passed color.
    • Provide a background color for an image conversion with transparency (alpha channel) to one that does not have it.
      • If not provided (kept as transparent), black (RGB 0, 0, 0) will be used.
  • It is not possible to replace the opaque background of an image with another of a different color.


Results Interface

After images are dragged or opened and the compression, resizing, and conversion process begins, the screen below appears. In it, you simply follow this process. The interface is divided into 6 parts, indicated by a number from 1 to 6. Just find the one that interests you.


1- List of processed images

Contains the images being processed for your review and monitoring. It includes the following fields:

  • A thumbnail, often showing only the center of the original image.
  • The file name.
  • Conversion indicator with the source and destination format.
  • An icon indicating whether the processing is local or remote.
  • Original size, compressed size, and compression percentage.
  • Processing time.
  • Process status:
    • Processing - animation
    • Successfully completed - green icon
    • An error occurred - red icon

2- Processing indicator

Visually indicates, as a percentage, how much of the processing has been completed.


3- Image counter

Indicates how many images are being processed and how many have finished.


4- Processing time

Indicates the total processing time for all images.


5- Actions

At this point, only the button to return to the main interface is displayed. Nothing prevents you from dragging new images to be processed onto this results interface. The app will indicate acceptance and process them in the same way.


6- Tinypng.com processing indicator

If there is a valid API key and it is enabled in the settings, it shows how many compressions were done in the month with that key. This makes it easy for you to control how many compressions you have already done.



Settings

Adjust the application settings here. The interface is divided into 4 parts, indicated by a number from 1 to 4. Just find the one that interests you.


1 - Tinypng API Key

If you choose to open a Tinypng account to access its powerful image compression and resizing API, enter your API key in this text box.


2- Forced conversions

The BMP, PCX, and HEIC formats are only supported for reading. Therefore, to be compressed and/or resized, they must be converted to another format. Here you choose this default format. Note that when you choose another conversion format in the main interface selector, it will take precedence over this setting.


3- Other settings

  • Always on top - Keeps the application window always on top of others.
  • Sound signal: Emits a sound signal at the end of processing.
  • Tinypng monthly count: Shows in the interface how many compressions were made using the Tinypng service in the current month.
  • Orientation correction: Attempts to correct the orientation of an image based on its EXIF data, if it exists and only in supported formats.
  • Reset all counters: Resets the compression counts made by the application and starts everything from zero again.

4- Command buttons

Save settings or return without saving.
