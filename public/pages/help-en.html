This app aims to facilitate the resizing and optimization of images. Its interface is almost entirely in the same window and the options are simple. The interface is divided into 6 parts, indicated by a number from 1 to 6. Just find the one that interests you.

1- File Box

Drop the image files you want to optimize in there or click the button to load files using your system's standard dialog box.


2- Image Adjustments

There are 5 possible adjustments described below:

  • Compression

    Defines where the image compression will come from.

    • Tinify when possible: Will always use the Tinipng service if it is online, has an API key and the image type is supported. Otherwise it will be local.
    • Best result: Will trigger 2 compression routines, one for Tinypng and another for the local one, provided that Tinypng meets the above requirements. Otherwise it will only be local. In the end, the image with the smallest size will be used.
    • Always local: It will always use the application's internal engine to perform all activities.

  • Background color

    Defines the background color for conversions to formats without an alpha channel (transparency). Currently only JPG. Choose the transparent color (checkerboard) to take advantage of the alpha channel. If the chosen color is transparent and the destination image has an alpha channel and is converted to JPG, black will be used as the background color.


  • Conversion

    Defines the format to which the resulting image will be converted. If none is chosen, the image remains in its original format. The exceptions are BMP, PCX, and HEIC, which are automatically converted to another format according to the configuration if a specific format is not chosen in this setting.


  • Files

    Defines how the new image will be created as a file. The options are:

    • Overwrite: The original file will be overwritten.
    • Create folder: A folder called optimized will be created, if it does not already exist, and the file will be created in it with the original name. If a file with the same name already exists in this folder, it will be overwritten.
    • New file: A new file with the suffix "-optimized" will be created.
    • Backup: Makes a backup copy of the original file with the suffix "-backup" before overwriting it.

  • Dimensions

    Defines the dimensions of the image being processed.

    • Do not change: The image will not be resized.
    • Scale: The image is resized on one of the axes. Either width or height. The choice appears when this option is selected. The new size in pixels and the axis it refers to. The size of the other axis is calculated to keep the image in the same aspect ratio.
    • Fit: The image will be resized to fit within a rectangle whose dimensions, width and height, are requested when this option is selected. The new size will be the largest possible to fit the chosen dimensions.
    • Cover: The image will be resized to cover the rectangle whose dimensions, width and height, are requested when this option is selected. The new size will be the smallest possible to cover the chosen dimensions. When used in Tinypng, AI is used to determine where to crop the image.
    • Thumbnail: Locally, it's the same as Cover. In Tinypng, it can stretch backgrounds to create thumbnails. Like Cover, width and height are requested.


3- Command Bar

The first 3 buttons allow you to check and uncheck options related to image metadata. They are, in order: Copyright information, geolocation, and image date.

The last 3, also in order, change the interface language, make a donation to the author, and invoke the Settings screen.


4- Counters

The optimizations made by this application are counted in this space. Both via Tinypng and locally. A general total and a monthly total are maintained.

The monthly count, both locally and online (Tinify), is only for compressions made using this app. Each image counts as one compression, regardless of conversions and/or resizing. To see how many compressions you've done in Tinify this month, enable "Show monthly Tinify compression count" in settings.


5- Totals

Shows the total saved and the number of optimizations made. The percentage refers to the total saved compared to the size before optimizations.
